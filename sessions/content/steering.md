# 作者如何思考与 steering

> 本章从全部 69 个 session 的作者原话中归纳。每条引文逐字保留（截断以 … 标注），并标注 session 与日期。一个背景事实：**这个项目用两种语言运转**——作者用中文指挥，产物（代码、文档、博客、推文）用英文发货。

先给全景观察：这位作者的 steering 语言极短、祈使、且**反常地具体**——几乎每个抽象要求都锚定在一个数字（32MB、60fps、333MHz）、一个具名范本（Tiptap、Linear、React DevTools、Roblox、XMB）、或一处他亲手实测得到的像素级观察上。

## 1 · 愿景到货时已完成 80%

立项消息有稳定的语法：动机 →（已经想好的）技术栈 → 第一个验证案例 → 未来的组织形态。技术选型是预先承诺的，但**证明义务用玩法而非功能清单来陈述**：

:::quote 2026-07-05 · S31
第一版不追求复刻经典 CS 手感，也不做联网或完整通用引擎，只需要证明：在 Pocket3D 里能加载 BSP 场景，第一人称走路，拿枪，遇到会移动和播放动画的简单 Bot，开枪击杀它，并完成一局胜负与自动重开循环。
:::

一句话可以立起一个产品并带上它的约束：

:::quote 2026-07-16 · S64
实现一个新的 pocket-youtube 应用，支持在 PSP 上看 youtube。它应该有配套的 macos 端宿主机服务为它解决网络协议层和像素流。不需要支持 wifi，仅支持 USB 插上 Mac 时使用即可
:::

而 demo 从来不只是 demo——它是引擎基建的强制函数：「做一个新的 figma viewer 的 example app……**借此完善 pocket 的引擎基建**」[[S47]]。当愿景本身是模糊的（paperworld），他会直说，并要求 agent 从第一性原理共同推导，而不是假装已有 spec。

## 2 · 砍需求发生在立项句里，不在评审会上

负空间与愿景同一口气写出，而且砍的都是**具体名词**：wifi、联网、手感、插图、断点——从不是空泛的"保持简单"。交付不达标的功能被优雅降级而非无限打磨：「首页那个 L 和 R 键可以直接隐藏掉，我觉得效果还不好，**我回头再改**」[[S66]]。MVP 由"必须可证明的东西"定义，一切不服务于证明的都被点名移除。

## 3 · 验收标准：真机 + 亲手拇指

验收线从来不是"测试通过"，而是"**我本人在真机上玩过并且它对了**"。「我实测」在语料里出现数十次；每次验收失败都升级为**基础设施**——harness、golden、确定性复现、一行命令：

:::quote 2026-06-21 · S8
没有，我实测仍然卡死了，我觉得你需要围绕 ppsspp 封装一个存档和发送操作的 harness……需要你完整搭出这套 infra 来确定性复现问题
:::

:::quote 2026-07-05 · S26
你需要给我封装一个一行的命令，我启动之后可以自己亲自体验和操作
:::

他甚至教测试方法论：「你最好搜 PSP，不要用 lofi 来测试」[[S66]]——因为静态的 lofi 流暴露不了视频运动 bug。验收达成时也会显式宣告：「很好，我现在觉得已经完全达到验收标准了。」

## 4 · 性能预算是道德事实

69 个 session 里仅有的两次爆粗，**都发生在性能或测量声明失去依据时**——不是代码写崩，而是认识论犯规：

:::quote 2026-06-17 · S2
你他妈在逗我吗？这肯定完全没有榨到极限，你认为就现在这种情况，能有什么极限？……你要不要看一看这台硬件是什么配置再说话？
:::

数字目标是单行军令（「car3d 需要推到 30」）。更深的模式是把体感的慢转换成**系统性问题**：不是"弄快点"，而是「是否有系统架构层面的优化可以继续挖掘出来？」，以及"慢是否应该静态可见"——后面这条思路直接催生了编译期烘焙和 capability 注册表：

:::quote 2026-07-12 · S55
我在权衡会不会出现业务样式写得差不多但突然有的地方就非常慢的情况……我在考虑这种样式是不是应该有一个 capability 之类的去表明它到底是怎么跑的。
:::

## 5 · 命名即架构

pocket-* 家族约定在一条消息里铸成：「它应该发布为 pocket-stack 组织下的 pocket-figma 项目。后续我们会继续实现 pocket-notion、pocket-linear 之类其他的项目，他们都遵守这个命名约定。」[[S47]]——**词法先于代码把路线图编码了**。品牌碰撞被预判（「很可能我还有类似 Notion 的那种 block，那怎么说呢？」）；CLI 改名不考虑向后兼容（「cli 命令从 pocketjs 改为 pocket 吧」）；口号连拼写与连字符都要立法（「Bare Metal Modern Web 我需要的是这么拼写」）。美学 steering 靠范本（XMB ribbon、open-design 配色）并在像素级核验；当"按描述迭代"失效时，他退回到一条不变式——「机身是多大圆角，shoulder button 就用多大圆角，完全一致，可不可以？懂不懂？」——这是他整个方法的微缩模型。

## 6 · 写作：hackathon 之声，不是 PR 稿

公共写作被三条硬规则治理。**其一，真实性**——每个技术最高级都要过审（「3D 动画直通 PSP GPU 硬件加速……这个表述准确吗，尤其是关于 3D 变换的部分，有编译期烘焙吗」），虚构的色彩当场处决：

:::quote 2026-07-11 · S47
然后 arguing about guidPath scoping at 2 a.m. 我不记得有这件事，也去掉
:::

**其二，语域**——工程直白、hackathon 兴奋、绝不公关腔：

:::quote 2026-07-11 · S47
包括你这个博客的开头，也有点太像是一篇 PR 稿了。实际上重点不应该是这个，而更应该像是一次 Hackathon 的那种成果展示的感觉。
:::

反装腔阀门在 VueConf 幻灯片周达到最大音量：「这份 PPT，本身也是 Vue 写的。 - 太装逼太恶心了！！！！！非常讨厌！」[[S65]]。**其三，情感必须是真的、他自己的**——他会把 agent 代写的情感段落换成真实的个人史（五年前读 Figma 多人协作论文的经历、Evan Wallace 是 role model），并坚持保留一句：「但是 with AI collaborator 是要有的」。

## 7 · 纠偏的烈度与认识论之罪成正比

崩溃的构建得到平静的「复现修看看」；没有依据的数字或错误前提才触发怒火。最凝练的否决是贴两张截图配一个词：「what」。每次纠偏都立即可执行——点名工件、附截图、给出可泛化的正确规则：

| 纠偏 | 教给 agent 的规则 |
|---|---|
| 「你那个每帧总工作量的统计是他妈什么鬼东西？……这根本就不是真实的东西」[[S65]] | 公开材料只允许**实测数字**；推算冒充数据是最重的罪 |
| 「不是，你理解是不是有一点本质的偏差？PocketJS 本来就支持用 Vue Vapor 来写东西」[[S65]] | 立论前先验证前提 |
| 「你直接 psplink 部署啊」[[S44]] | 常设基础设施就是拿来直接用的，不必请示 |
| 「怎么还没好，不应该等这么久的」[[S17]] | 他对任务时长有内在模型，超时的沉默本身就是 bug |
| 「是不是没有真的在烧 token 跑呢」[[S54]] | 声称"在跑"的自治系统要证明 LLM 调用是真实发生的 |

值得记录的细节：发火之后，他通常在一分钟内重发一条冷静的、规格化的同内容消息。怒火是信号，然后回到 spec。

## 8 · 工作流哲学：每个重复动作都变成工件

散文风格 → skill；发布流程 → skill；调试配方 → skill；规范 → CLAUDE.md 与记忆；验证 → harness。这是把**编译期烘焙的本能应用到协作本身**：付一次成本，永久复用。

:::quote 2026-07-05 · S24
你要把 PR 名字改成 conventional commit 的格式，然后也更新到你的记忆里面。另外，你看一下 agents.md，如果有这个事情的话，也要更新到那里面。
:::

授权按同心圆扩张：先「开一个 draft PR 就可以」，然后「自己合并进去」但要求先做一轮大的对抗性评审（「这个过程可以重复几次，直到确认没有问题以后再进行合并」[[S7]]），最终是跨仓库的常设自合并授权（「这些都是你自己开 PR 合并到主干就可以」[[S42]]）。连社区管理也带着判断力下放：「看一下 pr #71 值得搞吗，你判断一下，直接帮我去回复一下他」。

## 9 · 技术品味的四条轴

从原话中可提炼出一个稳定的品味系统：**专用胜于通用**（「不是想要做成 Unreal 那样大而全的通用引擎，而是特化出很多种小的专有引擎」）；**静态胜于动态**（凡编译期可知的——屏幕尺寸、样式、关键帧、图块、对白——一律烘焙）；**FFI 线神圣**（JS 编排、Rust 执行、每帧一次穿越，「文字渲染也立刻下沉 native」）；**capability 胜于假设**（功能要声明、匹配、opt-in，绝不默认存在——OSK 与光标的裁决都由此而来）。他用反事实竞争者推理（「我也不想直接就回退到使用原生的 Three.js，那样的话谁都可以做」），并把哲学句式当成真实的工程指令：「从本体论的视角出发」「从第一性原理出发」——期待 agent 从本质推导机制，而不是模式匹配现有框架。

## 10 · 最反常的模式：把 agent 当伙伴来 steering

语料里最不寻常的模式：作者把 agent 的偏好、难度估计和"工作条件"当作**真实的规划输入**。时间旅行调试就是这样被选中的：

:::quote 2026-07-07 · S42
1. 你觉得哪个对你来说是最难的？2. 哪个如果解决了，对你来说是最兴奋的？3. 哪个对你来说觉得是最空白、最值得探索的？
:::

DevTools 的立项书里两次出现「去做那些最能够满足你需要的功能」。里程碑之后有真正的复盘（「你觉得这次你的表现怎么样，你喜欢这个事情吗」[[S43]]），复盘得到的诉求随后被真金白银地排期：

:::quote 2026-07-17 · S64
上一轮这些我觉得 mac 副屏和硬解码可以稍微往后推一点……改善你的 DX 的目标我觉得更有价值，你讲讲
:::

他也主动邀请对自己心智模型的反驳：「告诉我：如果是你的话，你会更偏向于做成什么样？以及，我的理解大概有什么偏差？」[[S51]]

## 演化：六月的指挥官 → 七月中的合伙人

- **六月（dreamcart 期）**：命令语域。重型 ultracode 工作流、连发的「继续」、语料中最凶的纠偏。验证文化已在（ground truth、harness），但权限收紧：PR 只按明令开。
- **七月上旬**：立国语域。愿景、口号形而上学、Conventional Commits 与记忆纪律、首批发布、推文工坊。07-07 第一次出现对 agent 偏好的提问；07-08 授予常设自合并权。
- **七月中旬**：伙伴语域。product-per-day 的节奏、以分钟计的真机验收循环、问 agent 感受与愿望的复盘、agent-DX 优先于产品功能——同时文案标准收到最严（反装腔规则与假数据爆发都在 07-17）。
- **贯穿始终的常量**：真机是标尺、数字是锚点、demo 是引擎的强制函数、每个教训即时制度化。

## 最有人味的瞬间

高光时刻他用第一人称复数：「这次这个 CS 的 openstrike 移植是一次前无古人的成就。**我觉得我们一起解决了一个非常了不起的问题。**」[[S42]]。语料里最纯粹的玩家消息只有四个字：「我要真玩」。最物理的调试汇报只有三个字：「重插了」。对硬件的温柔（「我的 PSP 正在作为 U盘连接备份数据，所以千万不要做写操作」）、对自己致敬文字的羞涩（「Thank you, Evan 稍微有点不好意思」）、以及发布动画引擎时的 😈😈😈——情绪节拍永远拴在具体的物体上：一根重插的线缆、一个要过期的 token、一句坚持保留的致谢。
